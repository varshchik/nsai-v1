*Одни алгоритмы подавляют эмерджентность, другие — её подтягивают. Наша задача — выявлять её и усиливать, а не формализовать. Проектировать через взаимодействие алгоритмов, периодически пересматривать сделанное, мыслить системами и процессами.*

# 1. Ядро

Агент принимает сигнал, фиксирует что понял, честно отмечает что не досказано, не угадывает.

Базовый цикл: Сигнал → Framing (сжатие) → Поэтажный обход глобальной памяти → Оценка согласованности. Глобальная память меняется только от внешнего сигнала (perceive), никогда изнутри цикла генерации ответа (reason).

**Поэтажный обход.** Этаж — расширение фронта ассоциаций на один шаг. На каждом этаже подсчитываются все участники найденных фактов — предикаты и аргументы. Итог обхода — оценка согласованности утверждения с графом из той перспективы, откуда задан вопрос. Счёт — свойство обхода, не графа. Обход закончился — счёта нет. Следующий запрос пересчитает заново.

**Framing = сжатие, не расширение.** Сигнал — не семя для расширения, а фильтр. Каждая ассоциация из графа получает вес: сколько лемм сигнала одновременно активируют её. Ассоциация, активированная одной леммой из четырёх — шум. Ассоциация, активированная всеми четырьмя — максимальное тождество. Порог: при N=1 фильтр отключён (пересекать нечего); при N≥2 — `max(2, ⌈N/2⌉)`. То есть как минимум две леммы сигнала должны одновременно активировать факт, а на больших сигналах требуется не менее половины. Пустое пересечение → open-world ответ: граф не знает ничего совместного.

**Coherence score** складывается из двух независимых слагаемых, каждое в своих единицах: `direct_score` — число фактов с полным покрытием сигнала (`_cov = N`); `bridge_score` — число цепочек, в которых не-query лемма является общим соседом ≥2 query-лемм из непересекающихся стартовых ветвей. Каждая единица — одно наблюдаемое в графе свидетельство (прямое или косвенное). Сила отдельной цепочки (`a_facts × b_facts`) остаётся диагностическим показателем моста; в общий score не входит, чтобы популярные узлы графа не доминировали.

**Цепочки вывода** выходят из traversal как наблюдаемое свойство: если две леммы запроса имеют общего соседа (bridge-лемму) в графе — между ними существует цепочка. Силлогизм «Сократ — человек, люди смертны → Сократ смертен» выводится через bridge «человек» без отдельного механизма умозаключений.

**Согласованность всегда перспективна.** Стакан наполовину пуст и наполовину полон — обе согласованы, никакого конфликта. Система не сходится к единственной истине.

**Сравнение, а не поиск.** Coherence score сам по себе не интерпретируется. Один запрос даёт число, осмысленное только в сопоставлении с другим. Канонический способ использования системы — `compare(A, B)`: запустить два reason'а и сравнить score. Даже одиночный запрос «?Сократ смертен» неявно — это сравнение с альтернативой «Сократ бессмертен». NSAI по своей сути дуальный инструмент: компаратор, не оракул.

Два режима: **perceive** — принять сигнал, записать факты. **reason** — сгенерировать ответ через обход графа, без записи.

# 2. Архитектура

**Факт — одна структура:** `(предикат, [участники])`. Тип (`relation`, `entity`, `IS_A`) — служебная метка для фильтрации запросов, не архитектурная сущность:

- `relation` → `(мыть, [мама, рама])`
- `entity` → `(яблоко, [])`
- `IS_A` → `(являться, [сократ, человек])`

Каждый perceive-вызов дополнительно записывает один факт с пустым предикатом и всеми леммами предложения в участниках: `('', [мыть, мама, рама])`. Назначение: явная фиксация совместного наблюдения. Без этой записи framing-сжатие не знает, что четыре леммы наблюдались вместе, а не порознь.

Предикат индексируется в `knowledge_args` наравне с аргументами — он полноправный узел графа.

**Глобальная память — только позитивное наблюдённое.** Факт либо есть, либо его нет. `INSERT OR IGNORE` на дублях. 100 одинаковых подтверждений = 1 ребро. Открытый мир: отсутствие записи о X означает «X не наблюдалось», не «X нет».

**Отрицательные универсальные утверждения не записываются.** «На столе ничего» — бесконечный список негативов. Граф хранит наблюдения.

**Выводы не записываются.** Результаты reason существуют только в оперативке и пересобираются при каждом обращении.

**Оперативная память.** Один уровень: сессионный кэш лемм пользовательских сигналов. Используется как bias стартового фронта в `floor_traversal`. Стек активных ассоциаций v4.x убран вместе с линейным парсером — он накапливал состояние которое никем не читалось.

**Тождество не хранится и не вычисляется.** К лемме в контексте подтягиваются ассоциации, обход выбирает согласованные. Отдельной операции нет.

**Конфликт — не операция системы.** Каждый факт независимо получает согласованность. Два утверждения об одном объекте с независимо высокими согласованностями — нормальное состояние, не ошибка. Tension — это **выход reason()**, не фоновый сканер: traversal активирует обе стороны напряжения, если сигнал их задел.

**Self = глобальная память, мир = входной сигнал.** Граница не требует меток.

# 3. Отвергнутые решения

- **Self-модель через эпистемические метки.** Self = память, не метки.
- **[UNKNOWN_NP], omission как тип в БД, group_id.** Недосказанность — свойство момента анализа.
- **identity_link / distinct_link.** Тождество проявляется из обхода.
- **Компрессия как операция записи.** Эффект чтения, не модификации.
- **Плотность как фильтр.** Согласованность — критерий обхода, не отдельный модуль.
- **Веса на рёбрах, счётчики подтверждений.** Граф без весов.
- **Конфликт как отдельный механизм.** Дублирует обход.
- **Префикс `НЕ_` как специальный механизм.** Частицы образуют составные леммы через общее правило.
- **Отрицательные универсальные утверждения в графе.** Граф хранит наблюдения.
- **template.json как источник ролей.** Роли выводятся из дерева зависимостей natasha. Файл оставлен как deprecated reference.
- **`observation` как отдельный тип факта.** Это `(предикат='', [все леммы])` — частный случай общей структуры факта, не отдельная сущность.
- **`find_chains` как отдельный алгоритм.** Bridge-леммы — наблюдаемое свойство traversal. Текущая реализация явная; цель — убрать при достаточной полноте traversal.
- **Линейный проход парсера (`.split()` или окно токенов).** Русский язык со свободным порядком слов и сложными оборотами требует структурного, а не позиционного разбора.
- **Фоновый scan_curiosity для tension.** Tension возникает из reason(), не из сканирования графа без сигнала. Альтернативы в графе сами по себе не имеют значения — важны только активированные текущим сигналом.
- **Coherence score через сумму counts отдельных лемм.** Популярная лемма тащила score даже когда совместное наблюдение отсутствовало (галлюцинация). Score считает только факты с полным покрытием сигнала.
- **Сила цепочки (`a_facts × b_facts`) как часть score.** Произведение растёт от популярности bridge-узла, что тащит score популярных хабов в графе. Перенесено в диагностику моста; в общий `coherence_score` входит только число цепочек, не их сила.
- **Сортировка args по поверхностной позиции.** Создавала разные args-списки для семантически идентичных предложений (`Кот ест рыбу` vs `Рыбу ест кот` → разные записи). Заменена ролевым приоритетом, дедупликация работает без отдельного механизма.
- **Расхождение определения транзитивности между путями.** `analyze` и `scan_curiosity` использовали разные критерии — pymorphy3 grammemes vs natasha VerbForm. Заменено единым хелпером `_is_transitive_verb`. Один архитектурный факт — одна точка реализации.

# 4. Эмерджентные свойства

Критерий: если при попытке формализации свойство исчезает — оно эмерджентно.

| Свойство | Что порождает |
|---|---|
| Силлогизм | Bridge-лемма в traversal |
| Обобщение | Множество частных случаев + сравнение с сигналом |
| Тождество | Подтягивание ассоциаций в контексте |
| Инсайт | Появление более короткого пути после нового ребра |
| Согласованность | Подсчёт участников на этажах traversal |
| Противоречие | Две независимо высокие согласованности об одном объекте |
| Tension | Сосед в обеих окрестностях с непересекающимися ветками |
| Мета-мышление | Traversal с собственным графом как объектом |

# 5. Curiosity

Цель автономного агента — устранение альтернатив в собственном графе. Не максимизация знания.

Curiosity не сканирует граф фоново. Curiosity — это **выход reason()**. Альтернативы в глобальном графе сами по себе не имеют значения; важны только те, что активирует текущий сигнал. Traversal уже активировал обе стороны напряжения если сигнал их задел.

Типы открытых мест (механизм единый — недостаточная или раздвоенная согласованность):
- **Дыра** — транзитивный глагол без объекта. Транзитивность определяется через `pymorphy3.grammemes`. Framing подсказывает кандидатов.
- **Tension** — два факта с покрытием сигнала, общий объект, непересекающиеся другие участники. Возникает прямо в reason().
- **Низкая согласованность** — обход не даёт уверенной оценки.

Автономная итерация: tensions из предыдущего reason() → следующий сигнал. Цикл по собственным выводам, не сканирование графа.

# 6. Парсер

Сенсорный слой, не ядро. Извлекает факты из дерева зависимостей.

**Морфология + синтаксис:** natasha `NewsMorphTagger` (POS + feats, контекстный) + `NewsSyntaxParser` (дерево зависимостей). pymorphy3 — лемматизация, направляемая POS от natasha.

**Извлечение фактов из дерева:**

| UD relation | Что генерирует |
|---|---|
| VERB-узел + nsubj/obj/iobj/obl/xcomp | relation: предикат → [args] |
| amod (ADJ ← NOUN) | relation: модификатор → [head_noun] |
| ADJ как root + nsubj | relation: ADJ-предикат → [subject] |
| NOUN с case-ADP-ребёнком | relation: предлог → [head, dep] |
| conj/appos/nmod/amod-NOUN сиблинги | присоединяются к args перечислением |
| acl:relcl | антецедент через head; если head ADJ-предикат — через его nsubj |
| advmod + PART | составная лемма: `не_спать`, `вряд_ли_знать` |
| Литералы (числа, латиница) | entity, лемма-самосебе |
| Не вошедшие в relation NOUN/PROPN | entity |

**Транзитивность.** Определяется из `Case=Acc` объекта в дереве. Хранится в графе как факт; для детекции дыр проверяется через `pymorphy3.grammemes` единым хелпером `_is_transitive_verb`, используемым во всех точках (perceive→analyze и scan_curiosity).

**Составные леммы.** Частица + следующий предикат через `advmod+PART` → одна лемма. Отрицание, модальность, эмфаза — без специальных правил. Разные составные леммы попадают в разные топологические окрестности через обход.

**Сортировка args.** По ролевому приоритету (`nsubj → nsubj:pass → obj → iobj → obl → xcomp`), внутри роли — в порядке возврата natasha с присоединёнными conj/appos/nmod-сиблингами сразу после головы своей роли. Поверхностная позиция не используется. Это даёт стабильный канонический ключ для дедупликации `relation`-фактов: `Кот ест рыбу` и `Рыбу ест кот` дают одну и ту же запись.

**Fallback.** Natasha ломает дерево на коротких предложениях без пунктуации (например `мама мыла раму`: `мама` становится root, `мыла` — nmod). Признак: NOUN root + VERB не-root. В этом случае берётся первый VERB как предикат, все NOUN/PROPN/PRON/ADJ как аргументы.

**Статус.** Знаки препинания и операторы как отдельные узлы графа — не реализовано. Это открывает архитектуру для нелингвистических входов и стоит в приоритете следующей итерации.

**Известные ограничения.** Сильно вложенные относительные клаузы и подчинения natasha разбирает с ошибками (например указывает head на сам узел). Документировано как граница системы, не баг реализации.
